# Aushadham API Reference

**Last updated:** 2026-03-13
**Base URL (local):** `http://localhost:3000`
**Base URL (production):** Set via `BETTER_AUTH_URL` env var

---

## Overview

All API endpoints are served through a single Next.js catch-all route:

```
app/api/auth/[...all]/route.ts
```

This delegates all `/api/auth/*` requests to [Better Auth](https://www.better-auth.com/), which auto-generates the endpoints listed below based on the config in `lib/auth.ts`.

There are **no custom API routes yet**. Doctor data is served from a hardcoded mock in `lib/data/doctors.ts`.

## Authentication Endpoints

### Active (Used in UI)

| Method | Endpoint | Description | Request Body | Used In |
|--------|----------|-------------|-------------|---------|
| `POST` | `/api/auth/sign-in/email` | Sign in with email + password | `{ email, password, rememberMe? }` | `auth-modal.tsx` ‚Üí `signIn.email()` |
| `POST` | `/api/auth/sign-up/email` | Create a new account | `{ email, password, name, phone?, role? }` | `auth-modal.tsx` ‚Üí `signUp.email()` |
| `POST` | `/api/auth/sign-out` | Sign out, invalidate session | *(none)* | `navbar.tsx` ‚Üí `signOut()` |
| `GET` | `/api/auth/get-session` | Get current session & user data | *(none ‚Äî uses cookie)* | `navbar.tsx` ‚Üí `useSession()` |

## Authentication Details

### Session Management
- **Method:** Cookie-based (via `nextCookies()` plugin)
- **Expiry:** 7 days
- **Refresh:** Daily (sessions are refreshed every 24 hours on active use)

### User Schema (Custom Fields)
| Field | Type | Required | Default |
|-------|------|----------|---------|
| `name` | string | yes | ‚Äî |
| `email` | string | yes | ‚Äî |
| `phone` | string | no | ‚Äî |
| `role` | string | no | `"patient"` |

### Password Policy
- Minimum length: **8 characters**

### OAuth
- Google and GitHub client IDs are defined in `.env.local` but currently **empty/unconfigured**.

## Testing Endpoints

### Quick cURL Examples

**Sign Up:**
```bash
curl -X POST http://localhost:3000/api/auth/sign-up/email \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "testpass123",
    "name": "Test User"
  }'
```

**Sign In:**
```bash
curl -X POST http://localhost:3000/api/auth/sign-in/email \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "testpass123"
  }'
```

**Get Session** (pass the cookie from sign-in response):
```bash
curl http://localhost:3000/api/auth/get-session \
  -H "Cookie: better-auth.session_token=<token>"
```

**Sign Out:**
```bash
curl -X POST http://localhost:3000/api/auth/sign-out \
  -H "Cookie: better-auth.session_token=<token>"
```

## Database Tables

All managed by Drizzle ORM. Schema in `lib/schema.ts`, migrations in `drizzle/`.

| Table | Purpose | Key Columns |
|-------|---------|-------------|
| `user` | User accounts | id, name, email, phone, role, createdAt |
| `session` | Active sessions | id, token, userId (FK), expiresAt, ipAddress, userAgent |
| `account` | Auth providers (email/password, OAuth) | id, userId (FK), providerId, password |
| `verification` | Email verification & password reset tokens | id, identifier, value, expiresAt |

## API Documentation (Scalar)

**Status:** Installed and live

Interactive API docs are served at `/api/docs` (locally and on Vercel) using [Scalar](https://scalar.com) with the `@scalar/nextjs-api-reference` package.

### How it works

The docs are powered by two spec sources merged into one:

| Source | What it covers | How it stays in sync |
|--------|---------------|---------------------|
| **Better Auth `openAPI()` plugin** | All auth endpoints (sign-in, sign-up, sessions, password management, etc.) | Automatically ‚Äî any change to `lib/auth.ts` config or plugins updates the spec |
| **Manual spec in `app/api/openapi/route.ts`** | Custom business routes (`/api/doctors`, `/api/appointments`, etc.) | You add path entries here as you build new routes |

### Key files

| File | Purpose |
|------|---------|
| `lib/auth.ts` | Auth config ‚Äî includes `openAPI()` plugin that auto-generates auth spec |
| `app/api/openapi/route.ts` | Merges Better Auth's auto-generated spec with custom route specs, serves as JSON |
| `app/api/docs/route.ts` | Scalar UI ‚Äî reads from `/api/openapi` and renders the interactive docs page |

### Adding a new custom API route

1. Create the route (e.g., `app/api/doctors/route.ts`)
2. Add its OpenAPI path definition to the `customSpec.paths` object in `app/api/openapi/route.ts`
3. Visit `/api/docs` ‚Äî the new endpoint appears automatically

### Access

- **Local:** `http://localhost:3000/api/docs`
- **Production:** `https://yoursite.com/api/docs` (public for now, will be restricted later)
